## 🏗️ Step 1：创建项目（5 分钟）

### 1.1 选一个文件夹放项目

比如你在 `D:\` 下建一个 `Projects` 文件夹：

```powershell
cd D:\
mkdir Projects
cd Projects
```

### 1.2 用模板一键创建

我们用 **Tailwind Nextjs Starter Blog**（9k+ Stars，最成熟的 Next.js 博客模板）：

```powershell
npx create-next-app@latest my-blog -e https://github.com/timlrx/tailwind-nextjs-starter-blog
```

- 第一次用 `npx` 会问 `Ok to proceed? (y)`，输入 `y` 回车
- 等待下载（约 1-2 分钟）
- 看到 `Success!` 就 OK 了

### 1.3 进入项目 + 装依赖

```powershell
cd my-blog
npm install
```

这一步会装几百个依赖包，大约 2-5 分钟（国内网络可能慢，建议挂个代理或换 npm 镜像）。

> ⚠️ 如果 `npm install` 报错，试试换国内镜像：
>
> ```powershell
> npm config set registry https://registry.npmmirror.com
> npm install
> ```

### 1.4 跑起来！🎉

```powershell
npm run dev
```

看到类似这样的输出：

```
▲ Next.js 14.x.x
- Local:  http://localhost:3000
✓ Ready in 2.3s
```

**打开浏览器访问 `http://localhost:3000`**，恭喜你，博客已经跑起来了！🎊

按 `Ctrl + C` 可以停止服务。

```
│   ├── authors/            ← 作者信息
│   ├── projects.ts         ← 项目列表（作品集）
│   └── 📁 blog/            ← ⭐⭐⭐ 你的文章放这里！
```

## ✏️ Step 3：改成你自己的（20 分钟）

### 3.1 改站点信息

打开 `data/siteMetadata.js`，找到这些字段改成你的：

```js
const siteMetadata = {
  title: '你的博客名',                    // 比如 "XXX 的游戏开发笔记"
  author: '你的名字',                     // 比如 "张三"
  headerTitle: '你的博客名',
  description: '游戏程序/策划，分享独游开发、数值设计、技术文章',
  language: 'zh-cn',                     // 改成中文
  siteUrl: 'https://你的域名.vercel.app', // 先写个占位，部署后再改
  siteRepo: 'https://github.com/你的用户名/my-blog',
  siteLogo: '/static/images/logo.png',

  email: '你的邮箱@xxx.com',
  github: 'https://github.com/你的用户名',
  // twitter: 'https://twitter.com/xxx',  // 没有就注释掉
  // linkedin: '',

  locale: 'zh-CN',
  // ... 其他字段保持默认
}
```

**保存后**，浏览器里会自动刷新（如果你没停 `npm run dev` 的话），你会看到站点名字变了。这就是 Next.js 的 **热更新**——改代码，浏览器实时更新。

### 3.2 改作者信息

打开 `data/authors/default.mdx`，把里面的内容改成你自己的简介：

```mdx
---
name: 张三
avatar: /static/images/avatar.png
occupation: 游戏程序 / 独立开发者
company: 某某工作室
email: your@email.com
twitter: https://twitter.com/xxx
linkedin: https://linkedin.com/in/xxx
github: https://github.com/xxx
---

大家好，我是张三，一名游戏程序员/策划，专注于 **独立游戏开发**、**数值设计** 和 **引擎底层**。

这里是我记录折腾、踩坑、思考的地方。
```

> 💡 `---` 之间的部分叫 **Frontmatter**（元数据），下面的部分用 Markdown 写。这个文件会被渲染成 `/about` 页面。

### 3.3 换头像和 Logo

把你的头像图片放到 `public/static/images/` 目录下：

- 头像：`public/static/images/avatar.png`（建议 400x400）
- Logo：`public/static/images/logo.png`（可选，没有就用默认的）

然后在 `data/authors/default.mdx` 里引用的路径 `/static/images/avatar.png` 就能自动指向它。

> ⚠️ **注意**：Next.js 里引用 `public/` 下的文件，路径要 **去掉 `public/`**，直接从 `/` 开始写。

### 3.4 改导航栏

打开 `data/headerNavLinks.ts`，这里控制页面顶部的导航：

```ts
const headerNavLinks = [
  { href: '/', title: '首页' },
  { href: '/blog', title: '博客' },
  { href: '/tags', title: '标签' },
  { href: '/projects', title: '作品集' },
  { href: '/about', title: '关于' },
]

export default headerNavLinks
```

想加就加一行，想删就删一行，非常直观。

### 3.5 改作品集（可选）

打开 `data/projectsData.ts`，把里面的示例项目换成你自己的：

```ts
const projectsData = [
  {
    title: '我的独立游戏',

  },
  {

    imgSrc: '/static/images/tool.png',

  },
```

## 📝 Step 4：写你的第一篇文章（15 分钟）

### 4.1 删除模板自带的示例文章

打开 `data/blog/` 目录，你会看到一堆 `.mdx` 示例文件。你可以：

- **先别删**，拿它们当 MDX 语法参考
- 或者留一两篇作为格式范例，其余删掉

### 4.2 新建你的第一篇文章

在 `data/blog/` 下新建一个文件 `hello-world.mdx`（文件名就是 URL 的一部分，建议用 **英文-小写-连字符**）：

````mdx
---
title: 'Hello World：我的第一篇博客'
date: '2026-04-21'
tags: ['随笔', '开始']
draft: false
summary: '从今天起，我要在这里记录我的游戏开发之路。'
authors: ['default']
---

## 为什么开博客？

作为一个游戏程序，我发现很多踩过的坑、思考过的问题，过几个月就忘了。
所以决定开一个博客，把它们都记下来。

## 我会写什么？

- 🎮 独立游戏开发日志
- 🔢 数值设计 & 平衡性
- 💻 引擎底层 / Shader / 性能优化
- 📚 读书笔记 & 行业观察

## 一些代码示例

```csharp
// Unity C# 示例
void Update() {
    transform.position += Vector3.right * speed * Time.deltaTime;
}
```

> 欢迎来 [GitHub](https://github.com/xxx) 找我玩！
````

### 4.3 Frontmatter 字段说明

| 字段        | 说明                                           | 必填 |
| ----------- | ---------------------------------------------- | ---- |
| `title`   | 文章标题                                       | ✅   |
| `date`    | 发布日期（`YYYY-MM-DD`）                     | ✅   |
| `tags`    | 标签数组，用于分类                             | ❌   |
| `draft`   | `true` = 草稿（不公开），`false` = 发布    | ❌   |
| `summary` | 文章摘要（列表页展示）                         | ❌   |
| `authors` | 作者（对应 `data/authors/xxx.mdx` 的文件名） | ❌   |
| `images`  | 首图（社交分享时用）                           | ❌   |

### 4.4 预览

保存文件后，访问 http://localhost:3000/blog，你会看到新文章出现在列表里。点进去就是完整的文章页，自带目录、代码高亮、阅读时长估算、上下一篇导航——**全部自动生成**。

### 4.5 MDX 的超能力

MDX = Markdown + JSX。也就是说，你可以在文章里直接写 React 组件：

```mdx
import Card from '@/components/Card'

这是一段普通 Markdown。

<Card
  title="我的游戏"
  description="一个像素风 Roguelike"
  href="https://xxx"
/>

下面继续普通 Markdown。
```

这对游戏开发者来说太香了——**可以内嵌可交互的数值演示、可玩 demo、图表等**。

## 🎨 Step 5：改配色（可选，10 分钟）

> ⚠️ **注意**：这个模板用的是 **Tailwind CSS v4**，v4 **已经取消了 `tailwind.config.js` 文件**，主题配置全部搬进了 CSS 文件里的 `@theme` 块。所以你在项目里找不到 `tailwind.config.js` 是正常的。

模板默认是粉红色（pink）主题。想换成你喜欢的颜色？打开 **`css/tailwind.css`**，找到顶部的 `@theme { ... }` 块，里面有这样 11 行变量：

```css
@theme {
  /* Colors */
  --color-primary-50:  oklch(0.971 0.014 343.198);
  --color-primary-100: oklch(0.948 0.028 342.258);
  --color-primary-200: oklch(0.899 0.061 343.231);
  --color-primary-300: oklch(0.823 0.120 346.018);
  --color-primary-400: oklch(0.718 0.202 349.761);
  --color-primary-500: oklch(0.656 0.241 354.308);
  --color-primary-600: oklch(0.592 0.249   0.584);
  --color-primary-700: oklch(0.525 0.223   3.958);
  --color-primary-800: oklch(0.459 0.187   3.815);
  --color-primary-900: oklch(0.408 0.153   2.432);
  --color-primary-950: oklch(0.284 0.109   3.907);
}
```

这 11 个变量就是全站主题色。把它们整体替换成任意一种你喜欢的色系即可。

### 最简单的做法：从 Tailwind 官网复制一整套

1. 打开 https://tailwindcss.com/docs/colors
2. 挑一个你喜欢的颜色（比如 `emerald` 翠绿、`amber` 琥珀、`violet` 紫罗兰、`sky` 天蓝）
3. 点那一排色块下方的 `</>` 按钮，能直接复制 50~950 共 11 个 `oklch(...)` 值
4. 把它们粘到上面的 `--color-primary-*` 变量里即可

比如要改成 **翠绿色**：

```css
@theme {
  --color-primary-50:  oklch(0.979 0.021 166.113);
  --color-primary-100: oklch(0.950 0.052 163.051);
  --color-primary-200: oklch(0.905 0.093 164.150);
  --color-primary-300: oklch(0.845 0.143 164.978);
  --color-primary-400: oklch(0.765 0.177 163.223);
  --color-primary-500: oklch(0.696 0.170 162.480);
  --color-primary-600: oklch(0.596 0.145 163.225);
  --color-primary-700: oklch(0.508 0.118 165.612);
  --color-primary-800: oklch(0.432 0.095 166.913);
  --color-primary-900: oklch(0.378 0.077 168.940);
  --color-primary-950: oklch(0.262 0.051 172.552);
}
```

保存，浏览器自动刷新，主题色瞬间变了。

### 用十六进制色也行

如果你不习惯 `oklch()`，直接写 **HEX / RGB** 也完全可以：

```css
@theme {
  --color-primary-500: #ff6b6b;
  --color-primary-600: #ee5a5a;
  --color-primary-700: #dd4949;
  /* ... 其余类推 */
}
```

> 💡 **小技巧**：最常用的是 `-500`（主色）、`-600`（hover）、`-400`（暗色模式下的 hover）。改不动全套的话，先改这 3 个看效果。

## ☁️ Step 6：部署到 Vercel（10 分钟）

### 6.1 把代码推到 GitHub

先在 GitHub 上建一个新仓库（比如叫 `my-blog`），**不要勾选 README/gitignore/license**，保持空仓库。

然后回到项目目录：

```powershell
git init
git add .
git commit -m "initial commit"
git branch -M main
git remote add origin https://github.com/你的用户名/my-blog.git
git push -u origin main
```

刷新 GitHub 页面，代码就上去了。

> ⚠️ 如果 `git push` 让你输密码，注意 GitHub 现在 **不支持密码登录**，需要用 **Personal Access Token**（在 GitHub → Settings → Developer settings → Personal access tokens 生成）代替密码。

### 6.2 Vercel 一键部署

1. 打开 https://vercel.com，用 **GitHub 登录**
2. 点击 **Add New → Project**
3. 找到你刚推的 `my-blog` 仓库，点 **Import**
4. 配置页面**保持全默认**，直接点 **Deploy**
5. 等待 1-3 分钟，部署完成 🎉

你会拿到一个形如 `https://my-blog-xxx.vercel.app` 的网址，这就是你的博客线上地址。

### 6.3 回头更新 siteUrl

部署成功后，记得把 `data/siteMetadata.js` 里的 `siteUrl` 改成你真正的 Vercel 地址，然后：

```powershell
git add .
git commit -m "update siteUrl"
git push
```

Vercel 会自动重新部署。

### 6.4 绑定自定义域名（可选）

如果你有自己的域名（推荐在 Namesilo / Cloudflare 买 `.com` 约 70 元/年）：

1. Vercel 项目 → **Settings → Domains**
2. 输入你的域名，按提示去域名服务商那里加 CNAME 记录
3. 等几分钟生效，以后访问 `https://yourname.com` 就是你的博客了

## ❓ 常见问题速查

### Q1：`npm install` 一直卡住 / 报 network error

国内网络问题，换镜像或挂代理：

```powershell
npm config set registry https://registry.npmmirror.com
npm install
```

### Q2：`npm run dev` 报 `port 3000 already in use`

3000 端口被占了，换个端口：

```powershell
npm run dev -- -p 3001
```

### Q3：改了代码，浏览器没反应？

- 确认终端里 `npm run dev` 还在跑
- 硬刷新浏览器：`Ctrl + Shift + R`
- 实在不行，终端里 `Ctrl + C` 停掉，重新 `npm run dev`

### Q4：`.mdx` 文件改了但文章列表没更新？

检查 frontmatter 里：

- `draft` 是不是写成了 `true`（草稿不会显示）
- `date` 格式是不是 `YYYY-MM-DD`
- `tags` 要写成数组 `['a', 'b']`，不要写成字符串

### Q5：Vercel 部署失败？

打开 Vercel 项目 → **Deployments** → 点失败的那次 → 看 **Build Logs**，90% 的情况是：

- `data/siteMetadata.js` 里有语法错误（比如少了逗号）
- 某个 `.mdx` 的 frontmatter 格式错了
- 引用了不存在的图片

在本地跑一下 `npm run build`，本地能过 Vercel 基本就能过。

### Q6：文章里想插图怎么办？

1. 图片放进 `public/static/images/`（可以再建子目录，比如 `public/static/images/posts/hello-world/`）
2. MDX 里引用：

```mdx
![截图](/static/images/posts/hello-world/screenshot.png)
```

## 🎮 完成之后的下一步

恭喜！你已经有一个可以随时更新的个人博客了。作为**游戏程序/策划**，你还可以折腾这些：

### 进阶玩法

1. **嵌入可玩 Demo**
   把 Unity WebGL 或 Godot Web 导出的 `index.html` 塞进 `public/`，在文章里用 `<iframe>` 引用，就能在博客里直接玩你的游戏。
2. **数值演示组件**
   用 MDX 的特性，写一个 `<DamageCalculator />` 组件，让读者可以直接在文章里拖动 slider 看伤害公式变化。
3. **图表**
   接入 [Recharts](https://recharts.org) 或 [Chart.js](https://www.chartjs.org)，在文章里直接画数值曲线、经济系统图。
4. **Shader 在线预览**
   集成 [Shadertoy](https://www.shadertoy.com) 的 iframe，或者用 [twgl.js](https://twgljs.org) 写一个可交互的 WebGL demo。
5. **评论系统**
   模板已经内置 Giscus（基于 GitHub Issues），在 `data/siteMetadata.js` 里填上你的仓库和 repoId 即可开启。
6. **RSS / 邮件订阅**
   模板自带 RSS（`/feed.xml`）。想加邮件订阅？接入 [Buttondown](https://buttondown.email) 或 [ConvertKit](https://convertkit.com) 的 API。

### 学习路径建议

- 如果想**搞懂这套代码**：先学 React 基础 → Next.js App Router → 再回来看模板代码
- 如果只想**专心写作**：忽略以上，`data/blog/` 里敲键盘就完事了
- 如果想**做更酷的交互文章**：学一点 MDX + React 组件开发
